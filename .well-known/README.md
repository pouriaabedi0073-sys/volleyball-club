این پوشه برای نگهداری فایل‌های مربوط به استانداردهای well-known مانند `assetlinks.json` است.

نکات و روش‌ها:

1) چرا این پوشه؟
- گوگل و اندروید از مسیر `https://<your-domain>/.well-known/assetlinks.json` برای احراز مالکیت و برقراری ارتباط بین وب‌سایت و اپ اندروید استفاده می‌کنند.

2) ایجاد پوشه در ویندوز (در صورت وجود مشکل در Explorer):
- PowerShell:
  - `New-Item -ItemType Directory -Name '.well-known'`
  - یا: `mkdir .well-known`
- CMD:
  - `md .well-known`

3) آپلود در GitHub:
- می‌توانید از رابط وب گیت‌هاب یک فایل جدید بسازید و مسیر آن را به شکل `.well-known/assetlinks.json` وارد کنید؛ گیت‌هاب پوشه را به‌طور خودکار ایجاد می‌کند.
- اگر از GitHub Pages و پوشه `docs/` برای انتشار استفاده می‌کنید، پوشهٔ `.well-known` را داخل `docs/` قرار دهید تا پس از انتشار در ریشهٔ سایت قرار گیرد.

4) گزینهٔ جایگزین (در صورت سختی ایجاد پوشهٔ dot):
- قرار دادن فایل در مسیری دیگر و نوشتن قانون redirect/rewrite روی میزبان برای هدایت `/.well-known/assetlinks.json` به آن مسیر.
- یا استفاده از یک اسکریپت CI/GitHub Action برای کپی/تولید فایل در مسیر `/.well-known/` هنگام ساخت یا نشر.

5) نمونه و فایل مثال
- فایل نمونه `assetlinks.example.json` در همین پوشه وجود دارد؛ آن را با `package_name` و `sha256_cert_fingerprints` واقعی جایگزین کنید وقتی اطلاعات امضای اپ را دارید.

6) نکات امنیتی و محتوا
- مطمئن شوید فایل از طریق HTTPS و با Content-Type صحیح (`application/json`) در دسترس است.

اگر دوست دارید، من می‌توانم یک GitHub Action کوچک اضافه کنم که هنگام ایجاد یک تگ یا ریلیز، `assetlinks.json` را در شاخهٔ انتشار کپی کند (زمانی که شما مقادیر واقعی را فراهم کنید).
